# Requirements — Walking Simulator (Real World Edition)

## Видение

Веб-приложение в стиле Minecraft по структуре (бесконечный мир, чанковая генерация, биомы), но с **максимально возможным фотореалистичным качеством** в рамках браузера/WebGL2. Управление — с мобильного: джойстик слева снизу, свайп по экрану — поворот камеры, миникарта в правом верхнем углу с тапом-телепортом.

## Целевая платформа

- **Основная**: мобильный браузер (iOS Safari 16+, Android Chrome последний)
- **Вторичная**: десктоп Chrome/Firefox/Safari (с поддержкой клавиатуры/мыши)
- **Деплой**: статичный сайт (Vite build → GitHub Pages / Netlify / Vercel)

## Функциональные требования

### FR-1. Бесконечный процедурный мир
- **FR-1.1** Мир разбит на квадратные чанки (предлагаемый размер: 64×64 единиц).
- **FR-1.2** Чанки генерируются динамически по мере приближения игрока.
- **FR-1.3** Чанки за пределами `renderDistance` выгружаются для экономии памяти.
- **FR-1.4** Генерация детерминирована: один и тот же `seed` → один и тот же мир.
- **FR-1.5** Радиус видимости настраивается: `low=4 chunks`, `medium=8`, `high=12`.

### FR-2. Биомы (как в Minecraft, но реальные)
Минимум 6 биомов с плавными переходами:
- **FR-2.1** Поляна / лиственный лес (трава, дубы, цветы)
- **FR-2.2** Хвойный лес (сосны, ели, мох)
- **FR-2.3** Горы (скалы, камни, снежные вершины)
- **FR-2.4** Озеро / река (вода, песчаный берег, тростник)
- **FR-2.5** Степь / пустыня (трава, отдельные кусты, песок)
- **FR-2.6** Заснеженная тундра (снег, голые деревья)

Биомы определяются 2D-нойзами **температуры** и **влажности** (Whittaker-подход).

### FR-3. Рельеф
- **FR-3.1** Высота генерируется multi-octave Simplex noise (4–6 октав).
- **FR-3.2** Биом влияет на амплитуду/частоту (горы выше, поляна ровнее).
- **FR-3.3** Уровень воды — глобальная константа; всё ниже — затоплено.

### FR-4. Растительность и объекты
- **FR-4.1** Деревья размещаются по плотностной карте биома (poisson disk sampling или blue noise).
- **FR-4.2** Используется **GPU instancing** — миллионы травинок и тысячи деревьев без падения FPS.
- **FR-4.3** Камни, кусты, цветы — instanced billboards / меши с LOD.
- **FR-4.4** Трава реагирует на ветер (vertex shader displacement).

### FR-5. Реалистичный рендеринг
- **FR-5.1** PBR-материалы (albedo + normal + roughness + AO).
- **FR-5.2** HDRI окружение для image-based lighting (PolyHaven, CC0).
- **FR-5.3** Каскадные shadow maps (минимум 3 каскада).
- **FR-5.4** Реалистичная вода: отражения (планарные / cubemap), преломления, нормали, Fresnel.
- **FR-5.5** Объёмные облака (raymarched в небесном куполе или shader-based).
- **FR-5.6** День/ночь цикл (опционально для v2; в v1 — фиксированное "золотой час" освещение).
- **FR-5.7** Post-processing pipeline: ACES tone mapping, bloom, SSAO, TAA/FXAA.

### FR-6. Управление (мобильное-first)
- **FR-6.1** **Джойстик** в левом нижнем углу — движение вперёд/назад/влево/вправо.
  - Visible thumbstick на круглой основе (как nipplejs)
  - Чувствительность пропорциональна отклонению
- **FR-6.2** **Свайп по экрану** (где нет UI) — вращение камеры (yaw + pitch).
- **FR-6.3** **Pinch-to-zoom** (опционально) — изменение FOV.
- **FR-6.4** Десктоп: WASD для движения, мышь (pointer lock) для камеры.
- **FR-6.5** Камера от первого лица, высота персонажа ≈ 1.7m.
- **FR-6.6** Гравитация и коллизия с рельефом (игрок «стоит» на земле).

### FR-7. Миникарта
- **FR-7.1** Размер: ≈ 200×200 px в правом верхнем углу.
- **FR-7.2** Топ-даун вид окружающих чанков (≈ 16 чанков радиус).
- **FR-7.3** Цвета биомов отображаются (зелёный лес, серый горы, синий вода и т.д.).
- **FR-7.4** Стрелка/точка показывает игрока и направление взгляда.
- **FR-7.5** **Тап по миникарте → телепортация** в эту точку (высота подбирается по рельефу).
- **FR-7.6** Плавная анимация телепортации (fade out / fade in).

### FR-8. Производительность

**Целевой девайс**: Xiaomi Poco X6 Pro (Mediatek Dimensity 8300-Ultra, Mali-G615 MC6, 1220×2712 @120Hz, WebGL2/WebGPU).

- **FR-8.1** Целевой FPS на Poco X6 Pro: **стабильные 60 FPS** при High preset.
- **FR-8.2** Целевой FPS на десктопе: **60–120 FPS** (Ultra preset).
- **FR-8.3** Внутреннее разрешение рендера: **1080p**, апскейл до native через FSR1 / bilinear (экран 1.5K — нет смысла рендерить в native, экономим GPU).
- **FR-8.4** Адаптивное качество: 3 пресета (Low / High / Ultra). Автодетект девайса + ручной свитч. Если FPS падает < 45 на 3 секунды — авто-понижение пресета.
- **FR-8.5** Загрузка ассетов с прогрессбаром на старте; total < 30 MB.
- **FR-8.6** Объём JS-бандла без ассетов: < 500 KB gzipped.

## Нефункциональные требования

### NFR-1. Кодовая база
- TypeScript (strict mode).
- Vite как сборщик.
- Модульная архитектура (domain-driven, см. design.md).
- Без сторонних game-engine wrappers (чистый Three.js).

### NFR-2. Поддерживаемость
- Чёткое разделение: рендеринг ↔ генерация мира ↔ ввод ↔ UI.
- Каждый модуль самодостаточен и тестируем.

### NFR-3. Расширяемость
- Лёгкое добавление новых биомов (через конфиг).
- Лёгкое добавление новых моделей растительности.
- Готовность к будущим фичам: смена времени суток, погода, мульти-плеер.

### NFR-4. Доступность
- Работает без установки, прямо в браузере.
- Не требует разрешений (кроме fullscreen опционально).
- Отображает понятный fallback на устройствах без WebGL2.

## Что НЕ входит в v1 (out of scope)

- ❌ Разрушение/постройка блоков (это walking simulator, не sandbox)
- ❌ Мобы / NPC / животные
- ❌ Инвентарь / крафт
- ❌ Мультиплеер
- ❌ Сохранение прогресса (мир бесконечный и детерминированный — не нужно)
- ❌ Звук (можно добавить в v2: шаги, ветер, вода)
- ❌ Смена времени суток (фиксированное освещение в v1)

## Критерии приёмки

Запущенный сайт на телефоне должен:
1. ✅ Открыться < 5 секунд (включая загрузку ассетов).
2. ✅ Показать красивый стартовый вид с деревьями, травой, небом.
3. ✅ Дать ходить джойстиком и крутить камерой свайпом без багов.
4. ✅ Показать миникарту, которая обновляется при движении.
5. ✅ Телепортировать при тапе на миникарту.
6. ✅ Держать стабильные ≥ 30 FPS на телефоне 2022+ года.
7. ✅ Не «лагать» при пересечении границ чанков (бесшовная подгрузка).
